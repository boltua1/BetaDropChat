# 🚀 Гайд по запуску сервера BetaDropChat

Это краткое руководство поможет вам быстро настроить и запустить сервер для работы с Claude Code, Cursor и другими инструментами.

---

## 1. Подготовка (Один раз)
Перед первым запуском нужно «научить» сервер обходить защиту DeepSeek.

1.  Запустите меню управления:
    ```bash
    node scripts/auth.js
    ```
2.  Нажмите **1**, затем **Enter** (выбор аккаунта `default`).
3.  **В открывшемся окне Chrome:**
    *   Войдите в свой аккаунт DeepSeek.
    *   Отправьте любое короткое сообщение в чат.
    *   **Важно:** В консоли должна появиться надпись: `✅ Captured and saved pow.wasm`.
4.  Вернитесь в консоль и нажмите **Enter**. Браузер закроется сам.

---

## 2. Запуск сервера
Запустите главный файл:
```bash
node server.js
```
Вы увидите меню. Нажмите **4** (или просто **Enter**) для запуска.
*   Сервер заработает на `http://127.0.0.1:9655`.

---

## 3. Подключение к инструментам

### 🖥️ Claude Desktop (Интерфейс)
Вы можете использовать DeepSeek прямо в официальном приложении Claude Desktop:
1.  Откройте **Settings** -> **Inference**.
2.  Выберите **Gateway** в выпадающем списке.
3.  Настройте параметры (как на скриншоте):
    *   **Credential kind:** `Static API key`
    *   **Gateway base URL:** `http://127.0.0.1:9655`
    *   **Gateway API key:** Любые символы (например, `sk-ant-123`)
    *   **Gateway auth scheme:** `bearer`
4.  Нажмите **Test connection**.

### 🧩 Claude Code (CLI)
Просто запустите файл `run-claude.bat` в папке проекта. Он сам:
*   Запустит сервер в фоне.
*   Настроит мост между Claude и DeepSeek.
*   Откроет интерфейс Claude Code.

### 💻 Cursor / Open WebUI / Continue
Используйте следующие настройки в вашем приложении:
*   **API URL:** `http://127.0.0.1:9655/v1`
*   **API Key:** Любые символы (например, `sk-123`)
*   **Model:** `deepseek-chat` или `deepseek-reasoner` (для R1).

---

## 4. Как управлять аккаунтами
Если вы хотите использовать несколько аккаунтов DeepSeek:
1.  Снова запустите `node scripts/auth.js`.
2.  Выберите **1**, но вместо Enter введите имя (например, `work` или `boltuadev`).
3.  Пройдите авторизацию.
4.  Сервер сам подхватит новый аккаунт при следующем запуске.

---

## 🔍 Отладка и Логи
Если что-то идет не так:
*   **Логи запросов:** Находятся в папке `logs/ГГГГ-ММ-ДД/`. Там всё разложено по папкам аккаунтов.
*   **Файлы сессий:** Хранятся в `.deepseek/`. Если нужно «сбросить всё», просто удалите содержимое этой папки (кроме `pow.wasm`).
*   **Ошибка 403 / HTML:** Значит, DeepSeek обновил защиту. Просто повторите шаг №1 (Авторизация), чтобы обновить `pow.wasm`.
