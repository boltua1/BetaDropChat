# BetaDropChat API Proxy — Техническая документация

## Обзор

**BetaDropChat** — это локальный прокси-сервер, эмулирующий OpenAI и Anthropic API через веб-интерфейс **DeepSeek Web Chat** (`chat.deepseek.com`). Основная цель — обеспечить стабильную работу мультиагентных систем (например, Claude Code) с бесплатными моделями DeepSeek.

**Адрес сервера:** `127.0.0.1` (localhost)
**Порт по умолчанию:** `9655`
**Модели:** `deepseek-chat` (V3), `deepseek-reasoner` (R1), и их псевдонимы.

---

## 1. Архитектура и мультиаккаунтность

```
┌──────────────┐      POST /v1/messages         ┌──────────────────┐
│              │ ──────────────────────────────► │                  │
│  Multi-Agent │     {messages, model, stream}   │   BetaDropChat   │
│  (Clients)   │ ◄────────────────────────────── │   (Local Proxy)  │
│              │     Anthropic / OpenAI SSE      │   (Port 9655)    │
└──────────────┘                                 └────────┬─────────┘
                                                          │
                                         ┌────────────────┴────────────────┐
                                         │  • Multi-Account Routing        │
                                         │  • Session Lock (Concurrency)   │
                                         │  • Dynamic WASM/POW Solver      │
                                         └────────────────┬────────────────┘
                                                          │
                    ┌─────────────────────────────────────┼──────────────────────────────────┐
          ┌─────────▼─────────┐                 ┌─────────▼─────────┐              ┌─────────▼─────────┐
          │   DeepSeek Acc 1  │                 │   DeepSeek Acc 2  │              │   DeepSeek Acc N  │
          └───────────────────┘                 └───────────────────┘              └───────────────────┘
```

### 1.1 Управление сессиями (Concurrency Protection)
Сервер использует механизм **Session Lock** на уровне каждого агента. Если агент отправляет несколько параллельных запросов (например, Claude Code делает фоновые проверки), сервер гарантирует, что сессия в DeepSeek будет создана только один раз, а остальные запросы дождутся её готовности.

### 1.2 Мультиаккаунтность
Прокси может управлять несколькими аккаунтами одновременно. 
- **Маршрутизация**: Если указан конкретный `agent` в параметрах запроса, прокси подберет соответствующий `deepseek-auth-ID.json`.
- **Изоляция**: У каждого аккаунта свой профиль браузера и кэш сессий в папке `.deepseek/`.

## 2. Система логирования

Логи структурированы для удобного мониторинга мультиагентной активности:
- **Путь**: `logs/ГГГГ-ММ-ДД/ID_аккаунта/ID_агента.log`
- **Ключевые события**:
    - `INCOMING-REQUEST`: Входящий API запрос.
    - `SESSION-CREATED`: Успешное создание новой сессии в DeepSeek.
    - `OUTGOING-RESPONSE`: Финальная статистика ответа (время, токены).
    - `ERROR-*`: Подробные дампы ошибок (включая ответы DeepSeek).

## 3. API Endpoints

| Метод | Путь | Описание |
| --- | --- | --- |
| `GET` | `/v1/models` | Список поддерживаемых моделей |
| `GET` | `/v1/sessions` | Список активных сессий агентов |
| `GET` | `/v1/model-capabilities` | Маппинг моделей и их реальные возможности |
| `POST` | `/v1/chat/completions` | OpenAI Chat Completions API |
| `POST` | `/v1/messages` | Anthropic Messages API (для Claude Code) |
| `POST` | `/reset-session` | Сброс сессии (`?agent=ID` или `?agent=all`) |

## 4. Технические детали

### 4.1 Proof-of-Work (PoW) и WASM
Для обхода защиты DeepSeek используется WebAssembly модуль.
- **Авто-захват**: Скрипт `auth.js` перехватывает актуальный `.wasm` файл прямо из трафика Chrome во время логина.
- **Discovery**: Если файл отсутствует, `pow-worker.js` анализирует фронтенд DeepSeek для поиска актуальной ссылки.
- **Локальный кэш**: Файл хранится в `.deepseek/pow.wasm` для мгновенного доступа без сетевых задержек.

### 4.2 Tool Calling
Эмуляция инструментов происходит через инъекцию в системный промпт и парсинг ответов. Поддерживаются форматы как OpenAI (`tool_calls`), так и Anthropic (`tool_use`).
