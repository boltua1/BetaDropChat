# BetaDropChat API Proxy — Техническая документация

## Обзор

**BetaDropChat** — это локальный прокси-сервер, который эмулирует OpenAI и Anthropic API, используя веб-интерфейс **DeepSeek Web Chat** (`chat.deepseek.com`). Проект позволяет подключать бесплатные веб-модели DeepSeek к профессиональным инструментам разработки (Claude Code, Open WebUI, Cursor и др.).

**Адрес сервера:** `127.0.0.1` (localhost)
**Порт по умолчанию:** `9655` (настраивается в `settings.json`)
**Модели:** `deepseek-chat` (V3), `deepseek-reasoner` (R1), `deepseek-expert`.

## 2. Основные возможности

### 2.1 Изоляция сессий (Multi-Agent)
Каждый клиент (по IP или полю `user`) получает выделенную сессию в DeepSeek Web.
- **Session Lock**: Механизм предотвращения дубликатов чатов при параллельных запросах (актуально для Claude Code).
- **Auto-Recovery**: Автоматический сброс сессии при ошибках 401/403 или достижении лимита сообщений (>100).

### 2.2 Поддержка моделей
Прокси маппирует запросы на актуальные режимы DeepSeek Web:
- **`deepseek-chat`**: Быстрый режим (V3/V4-Flash).
- **`deepseek-reasoner`**: Режим рассуждений (R1). Передает `reasoning_content` в OpenAI формате.
- **`deepseek-expert`**: Режим повышенной точности (Expert).

### 2.4 Система логирования и диагностики
Прокси ведет подробные логи в папке `logs/`:
- **Структура**: `logs/YYYY-MM-DD/[agent_id].log`.
- **Типы событий**:
    - `INCOMING-REQUEST`: Полный входящий запрос (заголовки + тело).
    - `MODEL-MAPPING`: Информация о перенаправлении моделей.
    - `DS-REQUEST`: Запрос, отправленный на DeepSeek Web.
    - `ERROR-*-JSON`: Логирование некорректных ответов от DeepSeek для отладки.
- **Отображение рассуждений**: В не-стриминговом режиме `reasoning_content` автоматически вставляется в начало ответа в тегах `<thought>`.

## 3. Настройка и конфигурация

### 3.1 `settings.json`
Центральный файл настроек в корне проекта:
```json
{
  "auth_path": "./.deepseek/deepseek-auth.json",
  "chrome_profile_dir": "./.deepseek/chrome-profile",
  "server_port": 9655,
  "chrome_debug_port": 9334,
  "primaryModel": "deepseek-chat",
  "anthropicBaseUrl": "http://127.0.0.1:9655",
  "anthropicApiKey": "dummy-key"
}
```

### 3.2 Организация данных
Все временные и конфиденциальные данные хранятся в скрытой папке `.deepseek/`:
- `deepseek-auth.json`: Токены и куки.
- `chrome-profile/`: Профиль браузера для автоматизации логина.

## 4. API Endpoints

| Метод | Путь | Описание |
| --- | --- | --- |
| `GET` | `/health` | Статус сервера и готовность конфига |

## 5. Технические детали реализации

### 5.1 Proof-of-Work (PoW)
DeepSeek Web требует решения криптографической задачи для каждого сообщения. Прокси:
1. Загружает официальный WASM-модуль DeepSeek.
2. Решает задачу локально в рантайме Node.js.
3. Передает решение в заголовке `X-DS-PoW-Response`.

### 5.2 Tool Calling (Эмуляция)
Так как Web API не поддерживает инструменты нативно, BetaDropChat:
1. Инжектирует определения функций в системный промпт.
2. Парсит текстовый ответ модели на наличие паттернов `TOOL_CALL:`.
3. Преобразует их в структурированный JSON для клиента.

---

## 6. Запуск и отладка

Для детального анализа работы используйте `run-claude.bat` на Windows. Он запускает сервер в видимом окне, где логируются все входящие запросы и ответы от DeepSeek.

**Коды ошибок:**
- `502 empty_response`: DeepSeek вернул пустой ответ (требуется сброс сессии).
- `401/403`: Требуется обновление авторизации через `node scripts/auth.js`.
- `EADDRINUSE`: Порт 9655 занят другим процессом (лаунчер очистит его автоматически).
